# Локализованные страницы проектов

## 1. Контекст

Страницы проектов открываются по маршруту `/project/:projectId`. Компонент `ProjectPage` загружает данные из `/content/projects.ru.json` или `/content/projects.en.json`, а при ошибке использует fallback-данные из переводов.

Нужно описать поведение, которое гарантирует, что страницы проектов остаются доступными на русском и английском языках и корректно показывают детали проекта.

## 2. Цели

1. Пользователь открывает существующий проект по ID.
2. Страница показывает заголовок, описание, изображение, технологии и список возможностей.
3. При смене языка данные берутся из соответствующего локализованного JSON.
4. Если JSON недоступен или пустой, страница использует fallback-данные.
5. Для неизвестного проекта показывается состояние "проект не найден" и кнопка возврата.

## 3. Не-цели

- Не менять структуру публичных JSON-файлов.
- Не добавлять новый редактор проектов.
- Не проверять внешние сайты по demo-ссылкам.
- Не менять дизайн страницы проекта.

## 4. Пользовательские сценарии

S1. Пользователь открывает `/project/harmonize-me` и видит страницу проекта.

S2. Пользователь открывает проект после выбора русского языка и видит русские тексты.

S3. Пользователь открывает проект после выбора английского языка и видит английские тексты.

S4. Пользователь открывает неизвестный ID проекта и видит сообщение, что проект не найден.

S5. Пользователь нажимает кнопку возврата и возвращается назад по истории или на главную.

## 5. Бизнес-правила

- ID проекта должен быть стабильным и не зависеть от языка.
- Пустой или недоступный JSON не должен ломать страницу.
- Demo-кнопка показывается для проекта с `demoUrl`.
- GitHub-кнопка показывается только если `githubUrl` задан и не равен `#`.
- Первый Markdown-заголовок в `fullDescription` не должен дублировать заголовок страницы.

## 6. Критерии приёмки

- Для существующего проекта видны `h1`, описание, изображение и блок технологий.
- Для проекта с возможностями отображается список features.
- Для неизвестного проекта виден текст состояния "не найден" и кнопка возврата.
- При ошибке загрузки JSON страница остаётся работоспособной на fallback-данных.
- E2E или компонентные проверки покрывают минимум один существующий и один неизвестный проект.
