# План реализации

## Компоненты

- `src/pages/ProjectPage.tsx` — основная логика страницы проекта.
- `public/content/projects.ru.json` — русские данные проектов.
- `public/content/projects.en.json` — английские данные проектов.
- `src/locales/ru.json` и `src/locales/en.json` — fallback-переводы.
- `cypress/e2e/project-page.cy.ts` — возможная e2e-проверка.

## Основные решения

- Проверять поведение через публичный маршрут `/project/:projectId`.
- Для позитивного сценария использовать стабильный ID, например `harmonize-me`.
- Для негативного сценария использовать заведомо отсутствующий ID.
- Не проверять точные длинные описания, чтобы тест не ломался от редакторских правок.
- Проверять структурные элементы: заголовок, изображение, технологии, список возможностей, кнопки.

## Проверки качества

- Проверить загрузку страницы существующего проекта.
- Проверить страницу неизвестного проекта.
- Проверить, что fallback-данные не приводят к пустому экрану.
- Запустить сборку или e2e-проверку, если окружение позволяет.

## Ограничения

- Тексты проектов могут меняться через публичные JSON-файлы.
- Внешние demo-ссылки не должны открываться в тесте, чтобы не зависеть от сторонних сайтов.
