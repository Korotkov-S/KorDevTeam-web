# План реализации

## Компоненты

- `src/components/Blog.tsx` — список статей и карточки.
- `src/pages/BlogIndexPage.tsx` — страница `/blog`.
- `src/pages/BlogPostPage.tsx` — страница `/blog/:slug`.
- `cypress/e2e/blog-page.cy.ts` — e2e-спека.
- `openspec/specs/e2e-blog-navigation/spec.md` — спецификация поведения.

## Основные решения

- Использовать реальные видимые ссылки со страницы, а не жёстко зашитый список slug.
- Проверять первые несколько ссылок, чтобы сократить время выполнения.
- Каждый кейс начинать с повторного открытия `/blog`, чтобы сценарии не зависели друг от друга.
- Проверять именно пользовательский переход через `.click()`, а не прямой `cy.visit('/blog/<slug>')`.

## Проверки качества

- `openspec validate --all`.
- Локальный запуск Cypress, если зависимость и конфигурация доступны.
- При невозможности запуска Cypress зафиксировать блокер в итоговом описании.

## Ограничения

- Тест зависит от доступности live-сайта `https://kordev.team`.
- В `package.json` сейчас нет Cypress-зависимости и отдельного e2e-скрипта.
