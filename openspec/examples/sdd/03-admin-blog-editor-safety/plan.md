# План реализации

## Компоненты

- `src/pages/AdminPage.tsx` — UI редактирования блога.
- `server/routes/posts.js` — CRUD API постов.
- `server/routes/admin.js` — загрузка и удаление изображений.
- `server/middleware/auth.js` — Basic Auth, Bearer token и raw API key.
- `server/db/index.js` и `server/db/schema.js` — хранение постов.
- `server/utils/fileHandler.js` — legacy markdown-файлы.
- `server/utils/contentMeta.js` — извлечение метаданных из markdown.

## Основные решения

- API-методы создания, обновления и удаления должны проходить через `authenticate`.
- Валидацию обязательных полей держать на сервере, даже если UI тоже проверяет поля.
- Сохранять совместимость с legacy markdown-файлами, потому что статическая генерация блога зависит от файлов.
- Для удаления изображений использовать derivation ключа и проверку префикса `blog/uploads/`.
- Не расширять scope до ролей, RBAC или пользовательских аккаунтов.

## Проверки качества

- Unit-проверка генерации slug и нормализации языка.
- API-проверка отказа без авторизации.
- API-проверка создания поста с валидным payload.
- API-проверка отказа при пустых `title` или `content`.
- API-проверка запрета удаления изображения вне `blog/uploads/*`.

## Ограничения

- Не запускать миграции БД без явного согласования.
- Не удалять реальные production-файлы в тестах.
- Для тестов использовать временные данные и изолированное окружение.
