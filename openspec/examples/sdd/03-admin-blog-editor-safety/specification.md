# Безопасное редактирование статей в админке

## 1. Контекст

Админка позволяет редактировать блог и проекты. Для блога используются `AdminPage`, API `/api/posts`, middleware `authenticate`, SQLite-данные и markdown-файлы для обратной совместимости со статической сборкой.

Нужно описать безопасное поведение редактирования статьи: авторизация обязательна, обязательные поля валидируются, slug стабилен, изображения удаляются только из разрешённой области.

## 2. Цели

1. Администратор авторизуется через Basic Auth или Bearer/API key.
2. Администратор создаёт статью с заголовком и контентом.
3. Система сохраняет статью в базе и markdown-файле для статической сборки.
4. Администратор обновляет статью без потери coverUrl, tags, date и readTime.
5. Удаление cover-изображения разрешено только для путей `blog/uploads/*`.

## 3. Не-цели

- Не добавлять полноценную модель пользователей.
- Не реализовывать роли и права доступа.
- Не менять S3-интеграцию и storage-провайдеры.
- Не удалять изображения вне `blog/uploads/*`.
- Не запускать миграции базы без отдельного согласования.

## 4. Пользовательские сценарии

S1. Неавторизованный пользователь вызывает `/api/posts` с изменяющим методом и получает отказ.

S2. Авторизованный администратор создаёт статью с `title`, `content` и `lang`.

S3. Авторизованный администратор обновляет существующую статью и сохраняет её локализованную версию.

S4. Авторизованный администратор удаляет статью, после чего она исчезает из базы и legacy markdown.

S5. Администратор удаляет cover-изображение, и система принимает только безопасный ключ `blog/uploads/*`.

## 5. Бизнес-правила

- `title` и `content` обязательны при создании статьи.
- `lang` нормализуется через `safeLang`.
- Если slug не передан, он генерируется из заголовка.
- При обновлении существующей статьи slug не должен неожиданно меняться.
- Персональные данные и секреты авторизации не должны попадать в логи.
- Удаление файлов должно быть ограничено разрешёнными директориями.

## 6. Критерии приёмки

- POST `/api/posts` без авторизации возвращает 401 или 403.
- POST `/api/posts` без `title` или `content` возвращает 400.
- Успешное создание возвращает 201 и объект `post`.
- PUT `/api/posts/:slug` обновляет статью выбранного языка.
- DELETE `/api/posts/:slug?lang=...` удаляет статью выбранного языка.
- Попытка удалить изображение вне `blog/uploads/*` отклоняется.
