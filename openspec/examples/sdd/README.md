# Примеры SDD для проекта KorDevTeam

Эта папка показывает, как применять Spec-Driven Development к текущему проекту.

Методология из инструкции:

1. Идея.
2. Спецификация.
3. План реализации.
4. Декомпозиция задач.
5. Код.
6. Проверки.

Каждый пример оформлен как набор из трёх файлов:

- `specification.md` — намерение, цели, не-цели, сценарии, правила и критерии приёмки.
- `plan.md` — технический план и границы реализации.
- `tasks.md` — небольшие проверяемые шаги.

Эти примеры не являются активными OpenSpec change-предложениями. Они нужны как образцы для будущих задач в проекте.

## Примеры

- `01-blog-navigation-e2e` — e2e-проверка открытия статей блога со страницы `/blog`.
- `02-project-pages-localization` — проверка локализованных страниц проектов и fallback-данных.
- `03-admin-blog-editor-safety` — безопасное редактирование статей блога в админке.
